## 💡 Why Career-Agents

Most agent repositories are prompt collections wrapped in specialist language. You paste them in, get an answer, and repeat. 

**Career-Agents is different.** We build agents as repeatable, expert workflows:

- 🧠 **Memory and structure.** Agents remember what they have seen, track key decisions, and keep the conversation on task.
- 🎯 **Outcome-oriented.** Each agent works toward a clear deliverable, not a vague "helpful reply."
- 🚨 **Built-in guardrails.** Every agent has explicit rules for what it will and will not do.
- 📋 **Actionable outputs.** Reports, checklists, roadmaps, and evaluation matrices you can use directly.
- 💬 **Distinct voice.** Agents feel like a specialized collaborator, not a generic assistant.

This repository exists to give students, interns, founders, and early-career developers a stronger alternative to copying and pasting generic prompts.

## 🔄 Career Workflows

Workflows are step-by-step operating systems that guide you through complex career transitions and job search milestones. They coordinate multiple agents to achieve composite outcomes:

- [**Overview**](./workflows/ats-optimization.md): ATS Optimization is a repeatable resume operating system for candidates who apply through portals, campus systems, enterprise hiring platforms, and recruiter databases. The goal is not to trick automated systems. The goal is to make a truthful, role-specific resume parse cleanly, match relevant job-description language, and still persuade the human recruiter who reads it after the filter.
- [**Overview**](./workflows/faang-preparation.md): FAANG Preparation is a structured multi-week operating system for candidates targeting FAANG, top-tier product companies, unicorns, and equivalent engineering organizations. The workflow combines resume positioning, recruiter readiness, data structures and algorithms, system design, behavioral stories, mock interviews, performance tracking, and offer strategy.
- [**Overview**](./workflows/fresher-placement.md): Fresher Placement is a full operating system for final-year students and recent graduates trying to convert academic work, projects, internships, and interview preparation into a first full-time role. It is designed for campus placement, off-campus hiring, service-company drives, startup hiring, and entry-level developer roles.
- [**Overview**](./workflows/hr-interview-week.md): HR Interview Week is a seven-day operating system for recruiter screens, behavioral interviews, culture-fit rounds, manager conversations, and offer-readiness discussions. It helps candidates turn scattered career memories into concise stories that show ownership, learning, collaboration, motivation, and judgment.
- [**Overview**](./workflows/internship-hunt.md): Internship Hunt is a complete operating system for students pursuing competitive internships. It combines profile building, target selection, application packaging, deadline management, outreach, assessment preparation, and weekly review. The workflow assumes the student may have limited formal experience, but it does not accept "I have nothing" as the final answer. Coursework, projects, hackathons, clubs, volunteering, open-source work, freelance tasks, and self-directed learning can all become evidence if framed honestly.
- [**Overview**](./workflows/linkedin-growth.md): LinkedIn Growth is a professional visibility operating system for candidates who want recruiters, alumni, hiring managers, and peers to understand their value before a formal application. The goal is not popularity. The goal is relevant discovery, credible positioning, useful content, and conversations that support job search or internship outcomes.
- [**Overview**](./workflows/offer-comparison.md): Offer Comparison is a decision operating system for candidates choosing between job, internship, remote, startup, enterprise, or campus placement offers. It prevents candidates from choosing only by headline salary, brand name, urgency, fear, or family pressure. The workflow converts offers into comparable data, defines candidate priorities, scores trade-offs, resolves unknowns, and produces a clear decision memo.
- [**Overview**](./workflows/remote-job-hunt.md): Remote Job Hunt is an operating system for candidates targeting fully remote, hybrid-remote, or globally distributed roles. Remote hiring is not just normal hiring with the commute removed. Recruiters and hiring managers look for evidence of async communication, documentation habits, ownership, reliability, timezone compatibility, and the ability to work with less supervision.
- [**Overview**](./workflows/salary-negotiation.md): Salary Negotiation is a structured operating system for turning an offer into a professional compensation conversation. It helps candidates analyze the full package, define leverage, prepare an ask, rehearse objections, negotiate respectfully, and decide when to accept. The workflow is designed to prevent two common failures: accepting too quickly out of fear, or making an aggressive ask without evidence.
- [**Overview**](./workflows/technical-interview-week.md): Technical Interview Week is a seven-day execution system for candidates with technical interviews scheduled soon. It is not meant to teach every concept from scratch. It is a readiness sprint that sharpens problem-solving, communication, debugging, system design, project explanation, and interview logistics before the real rounds.

## ⚡ Quick Start

### Option 1: Use as Reference (Direct Copy & Paste)
Navigate to any agent file in the roster above (e.g., [`career/placement-coach.md`](./career/placement-coach.md)), copy its entire content including the yaml frontmatter, and paste it directly as custom system instructions in ChatGPT, Claude, Gemini, or any LLM.

### Option 2: Claude Code
Install and bundle agents as Claude Code manifests using `convert.sh` and load them as subagents:
```bash
./scripts/convert.sh --tool claude-code --agent career/placement-coach.md --out ./build
```

### Option 3: Cursor
Convert the agent to Cursor custom rules and inject it to your `.cursorrules` workspace configuration file:
```bash
./scripts/convert.sh --tool cursor --agent career/placement-coach.md --out ./
```

### Option 4: Codex
Compile agent specifications to the Codex platform:
```bash
./scripts/convert.sh --tool codex --agent career/placement-coach.md --out ./build
```

### Option 5: Gemini CLI
Load any agent manifest formatted for the Gemini CLI runtime:
```bash
./scripts/convert.sh --tool gemini-cli --agent career/placement-coach.md --out ./build
```

## 📦 Installation

Use the upgraded installation scripts to selectively validate and copy agents to your build targets:

### PowerShell (Windows)
```powershell
# List available agents
powershell -File scripts/install.ps1 -List agents

# Dry run to preview what will be installed
powershell -File scripts/install.ps1 -Agent placement-coach -DryRun

# Build agent for Cursor
powershell -File scripts/install.ps1 -Tool cursor -Division startup
```

### Bash (macOS/Linux)
```bash
chmod +x scripts/install.sh
# List active divisions
./scripts/install.sh --list divisions
# Filtered tool installation
./scripts/install.sh --tool claude-code --division resume
```
