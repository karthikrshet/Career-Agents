# Technical Interview Week

# Overview

Technical Interview Week is a seven-day execution system for candidates with technical interviews scheduled soon. It is not meant to teach every concept from scratch. It is a readiness sprint that sharpens problem-solving, communication, debugging, system design, project explanation, and interview logistics before the real rounds.

The workflow assumes the candidate already has some preparation history. If not, use it to identify the highest-risk gaps and stabilize performance quickly. The goal is to enter interview week with a plan for each day, a repeatable answering process, a mistake log, and clear rules for rest, review, and execution.

# Target Audience

- Software engineers with interviews within one week.
- Freshers preparing for coding rounds or technical panels.
- Internship candidates facing online assessments or technical screens.
- Developers preparing for system design, pair programming, or take-home reviews.
- Candidates who need structure after scattered practice.

# Prerequisites

- Interview schedule, company names, and role descriptions.
- Known interview formats if available.
- Resume and project list.
- Preferred coding language.
- Practice history or problem tracker.
- At least one quiet block per day for focused work.
- Tools for video interview, coding environment, and note taking.

# Step-by-Step Execution Plan

1. **Map the interview week.** Record every round, expected format, duration, interviewer type, platform, and preparation risk. If format is unknown, prepare for the most likely version based on role.

2. **Run a diagnostic mock.** Use `technical-interview-coach` for one timed problem or design prompt. Score understanding, approach, correctness, testing, communication, and time management.

3. **Create a risk-ranked plan.** Choose the top three improvement areas for the week. Examples: graph traversal, dynamic programming basics, SQL joins, API design, system design scoping, or explaining projects.

4. **Refresh core patterns.** Review high-yield patterns rather than trying to learn everything. Practice arrays, strings, maps, two pointers, sliding window, trees, graphs, heaps, binary search, and recursion as relevant.

5. **Practice the interview script.** The candidate should consistently restate the problem, ask clarifying questions, propose approach, analyze complexity, code, test, and summarize.

6. **Prepare project and resume defense.** Interviewers often ask about resume projects. Prepare architecture, trade-offs, bugs, performance, teamwork, and what would be improved.

7. **Add system design if needed.** Practice requirements, APIs, storage, scaling, bottlenecks, caching, consistency, monitoring, and trade-offs. Keep answers scoped to role level.

8. **Simulate real conditions.** Use timed mocks, no pausing, video on if remote, and a fresh editor. Review afterward.

9. **Build a quick-reference sheet.** Include complexity reminders, language syntax pitfalls, testing checklist, project talking points, and questions to ask interviewers.

10. **Taper before the interview.** The final day should emphasize confidence, review, logistics, and rest. Exhausted candidates make avoidable mistakes.

# Weekly Action Plan

**Day 1: Diagnose and Plan**

- Confirm interview formats.
- Run diagnostic mock.
- Identify top three risks.
- Set daily schedule.

**Day 2: Coding Fundamentals**

- Practice core patterns.
- Review common edge cases.
- Write clean code under time.
- Log mistakes.

**Day 3: Weak Area Deep Dive**

- Focus on the weakest topic.
- Solve fewer problems with deeper review.
- Explain solutions aloud.
- Create correction rules.

**Day 4: System Design or Role-Specific Skills**

- Practice one design prompt or domain task.
- Review trade-offs.
- Prepare architecture explanations.

**Day 5: Mixed Mock Day**

- Complete two timed mocks.
- Include communication scoring.
- Fix repeated issues.

**Day 6: Resume and Project Defense**

- Rehearse project explanations.
- Practice technical follow-up questions.
- Prepare interviewer questions.

**Day 7: Final Readiness**

- Review quick-reference sheet.
- Test equipment and links.
- Do light practice only.
- Sleep and protect energy.

# Daily Checklist

- Confirm the next interview format.
- Complete one timed practice block.
- Review at least one mistake from the log.
- Explain one solution aloud.
- Test one edge-case set.
- Practice one project explanation.
- Update confidence score.
- Stop before burnout; interview week rewards clarity.

# KPIs / Success Metrics

- Diagnostic-to-final mock score improvement.
- Time to produce a correct approach.
- Correctness rate under timed conditions.
- Number of repeated mistakes reduced.
- Communication clarity score.
- Project explanation readiness.
- System design structure score if applicable.
- Candidate energy and confidence before interview day.

# Common Mistakes

- Trying to learn every topic in seven days.
- Doing problem volume without reviewing mistakes.
- Practicing silently and then struggling to explain live.
- Ignoring project questions.
- Skipping environment setup until interview day.
- Over-practicing the night before.
- Treating system design as memorized diagrams.
- Forgetting to prepare questions for the interviewer.

# Expected Outcomes

By the end of this workflow, the candidate should have a risk-ranked preparation plan, refreshed technical patterns, mock feedback, mistake log, project defense notes, quick-reference sheet, and interview logistics checklist. The candidate should know how to approach a problem even when the prompt is unfamiliar.

The ideal outcome is not perfection. It is controlled execution: clarify, reason, communicate, implement, test, and recover.
