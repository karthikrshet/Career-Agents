# FAANG Preparation

# Overview

FAANG Preparation is a structured multi-week operating system for candidates targeting FAANG, top-tier product companies, unicorns, and equivalent engineering organizations. The workflow combines resume positioning, recruiter readiness, data structures and algorithms, system design, behavioral stories, mock interviews, performance tracking, and offer strategy.

This is not a cram sheet. Elite technical hiring processes reward consistent problem-solving, clear communication, ownership stories, and the ability to reason under pressure. The workflow is designed as a six- to twelve-week preparation cycle. Candidates with interviews already scheduled can compress it, but they should not skip diagnosis and feedback loops.

# Target Audience

- Software engineers targeting FAANG or equivalent companies.
- Students preparing for elite internship or new-grad technical rounds.
- Developers moving from smaller companies to large-scale product engineering.
- Candidates preparing for coding, system design, and behavioral loops.
- Contributors documenting advanced technical interview workflows.

# Prerequisites

- Target company list and role level.
- Resume and LinkedIn profile.
- Baseline coding ability.
- Knowledge of one interview programming language.
- Calendar availability for daily practice.
- Tracker for problems, mocks, feedback, and scores.
- List of interviews scheduled or expected timelines.

# Step-by-Step Execution Plan

1. **Define target level and interview loop.** New-grad, L3, L4, L5, senior, and staff loops are different. Record expected rounds: recruiter screen, coding, system design, behavioral, manager, domain, or bar raiser.

2. **Audit resume for top-tier signal.** Use `resume-strategist` to highlight impact, scale, technical depth, ownership, and measurable outcomes. Large companies need evidence that maps to role level.

3. **Set a baseline mock.** Run one timed coding problem and one communication review. Senior candidates should also complete one system-design diagnostic. Record scores before studying.

4. **Build the topic map.** Cover arrays, strings, hash maps, stacks, queues, linked lists, trees, graphs, heaps, binary search, recursion, dynamic programming, greedy, intervals, and concurrency or language-specific topics if relevant.

5. **Practice by pattern, not random panic.** For each topic, learn the pattern, solve easy and medium problems, review mistakes, then do timed mixed practice.

6. **Create a mistake log.** Track wrong assumptions, edge cases missed, time overruns, syntax issues, unclear explanation, and repeated pattern gaps.

7. **Add system design.** For mid-level and senior roles, practice requirements, APIs, data model, bottlenecks, scaling, caching, consistency, monitoring, and trade-offs.

8. **Build behavioral story bank.** Prepare stories for conflict, failure, leadership, ambiguity, ownership, mentoring, technical trade-offs, and impact. Use `hr-interview-coach`.

9. **Run mock interviews weekly.** Use timed conditions. Score understanding, approach, correctness, code quality, complexity, testing, and communication.

10. **Prepare company-specific plans.** For each target company, note role expectations, likely interview emphasis, recruiter communication, and questions to ask.

11. **Simulate the final loop.** Before real onsite or virtual onsite, run a full mock sequence: coding, design, behavioral, break, coding.

12. **Prepare offer strategy.** If multiple companies are active, plan timelines and decision criteria early.

# Weekly Action Plan

**Week 1: Baseline and Setup**

- Resume refresh.
- Baseline coding mock.
- Topic map and schedule.
- Mistake log setup.
- Behavioral story inventory.

**Weeks 2-3: Core Coding Patterns**

- Daily DSA practice.
- Pattern review.
- Timed mediums.
- Weekly mock.
- Mistake log review.

**Weeks 4-5: Mixed Practice and Communication**

- Mixed timed sets.
- Explain solutions aloud.
- Practice debugging.
- Behavioral story rehearsal.
- Recruiter outreach or application push.

**Weeks 6-7: System Design and Full Mocks**

- System design drills.
- Architecture trade-off practice.
- Full loop simulation.
- Company-specific preparation.

**Week 8+: Interview Execution**

- Reduce volume.
- Focus on weak patterns.
- Review stories and questions.
- Manage recruiter timelines and offer strategy.

# Daily Checklist

- Solve at least one timed problem.
- Review one previous mistake.
- Explain one solution aloud.
- Practice one edge-case analysis.
- Spend 20-30 minutes on system design if level requires it.
- Rehearse one behavioral story.
- Update problem tracker and confidence score.
- Rest deliberately; exhausted practice produces false learning.

# KPIs / Success Metrics

- Median time to solve medium problems.
- Correctness rate under timed conditions.
- Number of patterns mastered.
- Repeat mistake reduction.
- Mock interview rubric score.
- System design completeness score.
- Behavioral story readiness.
- Recruiter screen conversion.
- Onsite or final-loop invitations.

# Common Mistakes

- Solving many problems without reviewing mistakes.
- Memorizing solutions instead of recognizing patterns.
- Ignoring communication until real interviews.
- Practicing only coding when the role requires system design.
- Over-indexing on hard problems before fundamentals are stable.
- Not preparing behavioral stories for ownership and failure.
- Scheduling interviews before readiness is measured.
- Treating company prestige as the only decision criterion.

# Expected Outcomes

By the end of this workflow, the candidate should have a top-tier resume, a structured problem tracker, a mistake log, improved coding speed, system-design practice artifacts, behavioral story bank, mock scores, and a target-company execution plan. The candidate should know their readiness level rather than guessing.

The workflow is successful when interview performance becomes repeatable: the candidate clarifies, proposes, codes, tests, explains, and recovers from mistakes with discipline.
