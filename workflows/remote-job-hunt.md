# Remote Job Hunt

# Overview

Remote Job Hunt is an operating system for candidates targeting fully remote, hybrid-remote, or globally distributed roles. Remote hiring is not just normal hiring with the commute removed. Recruiters and hiring managers look for evidence of async communication, documentation habits, ownership, reliability, timezone compatibility, and the ability to work with less supervision.

This workflow helps candidates define the right remote role, build remote-specific positioning, find remote-friendly companies, prepare for asynchronous assessments, verify work conditions, and avoid roles that advertise remote flexibility but operate with hidden office expectations. It is especially useful for software developers, designers, product people, support specialists, and early-career candidates targeting distributed teams.

# Target Audience

- Software developers seeking remote-first roles.
- Candidates applying across cities, countries, or time zones.
- Professionals moving from office work to distributed teams.
- Freshers who need to prove remote reliability.
- Candidates evaluating remote offers and work conditions.
- Contributors documenting remote career workflows.

# Prerequisites

- Target role and seniority level.
- Resume and LinkedIn profile.
- Portfolio, GitHub, writing samples, or proof of independent work.
- Preferred timezone overlap and location constraints.
- Remote work history if available.
- List of remote job boards, companies, or communities.
- Application tracker.

# Step-by-Step Execution Plan

1. **Define remote requirements.** Decide whether the candidate wants fully remote, hybrid, remote within country, remote across time zones, contractor, full-time employee, or async-first work. Record non-negotiables.

2. **Clarify timezone fit.** Define preferred working hours, maximum meeting overlap, relocation willingness, and legal work location. Many remote roles fail because this is not clarified early.

3. **Build remote-readiness evidence.** Identify examples of independent work, written communication, documentation, async collaboration, project ownership, open-source participation, distributed teamwork, or self-managed delivery.

4. **Update resume and LinkedIn.** Add remote-relevant signals without forcing them. Examples: documented APIs, wrote onboarding notes, collaborated across time zones, shipped features independently, maintained project READMEs, contributed to distributed teams.

5. **Create the remote company map.** Separate remote-first companies, remote-friendly companies, hybrid companies, contractor marketplaces, open-source ecosystems, and communities. Prioritize companies with explicit remote policies.

6. **Build application channels.** Use remote job boards, company pages, LinkedIn, GitHub communities, Slack/Discord communities, alumni, recruiter posts, and founder-led hiring channels.

7. **Write remote-specific outreach.** Messages should mention role fit and remote operating evidence, not just "I want remote." Recruiters care whether the candidate can succeed remotely.

8. **Prepare for remote assessment formats.** Remote processes often include take-home projects, recorded screens, async written answers, pair programming, system design over video, and communication-heavy interviews.

9. **Verify remote reality.** During interviews, ask about meeting expectations, overlap hours, documentation culture, onboarding, equipment, performance review, contractor vs employee status, benefits, and location restrictions.

10. **Evaluate offers carefully.** A remote role may have lower commute cost but higher isolation, timezone strain, legal ambiguity, or weak onboarding. Use `offer-evaluation-advisor`.

# Weekly Action Plan

**Week 1: Remote Positioning**

- Define remote constraints.
- Rewrite resume and LinkedIn for remote evidence.
- Build remote proof section in portfolio or GitHub.
- Create first list of 40 remote-friendly companies.

**Week 2: Pipeline Launch**

- Submit 15 targeted remote applications.
- Send 10 remote-specific outreach messages.
- Join 2-3 relevant communities.
- Prepare async writing sample or project explanation.

**Week 3: Assessment Preparation**

- Practice one take-home simulation.
- Run one video technical mock.
- Prepare written answers for remote collaboration questions.
- Follow up with recruiters and contacts.

**Week 4: Review and Verification**

- Calculate response rate by channel.
- Remove low-quality remote listings.
- Prepare interview questions about remote culture.
- Evaluate active opportunities for timezone and legal fit.

# Daily Checklist

- Review one remote-friendly company.
- Apply to or follow up on 2-3 quality roles.
- Send one targeted remote outreach message.
- Improve one remote-readiness proof point.
- Practice written communication: explain a project decision clearly.
- Update tracker with timezone, location rules, and status.
- Avoid roles with unclear remote policy until clarified.

# KPIs / Success Metrics

- Remote-specific applications submitted.
- Response rate from remote-friendly companies.
- Outreach reply rate.
- Interviews in acceptable timezone windows.
- Number of opportunities with verified remote policy.
- Take-home or async assessment readiness.
- Offer alignment with timezone, employment status, and flexibility needs.
- Reduction in applications to fake-remote or unclear roles.

# Common Mistakes

- Applying to remote roles with no evidence of remote work habits.
- Ignoring timezone overlap until late interviews.
- Assuming "remote" means "work from anywhere."
- Using generic outreach that does not mention remote readiness.
- Underestimating written communication in remote hiring.
- Accepting contractor terms without understanding benefits, taxes, or stability.
- Failing to verify onboarding and manager support.
- Treating remote work as a perk instead of an operating model.

# Expected Outcomes

By the end of the workflow, the candidate should have remote-ready application materials, a company and channel map, outreach templates, assessment preparation plan, remote culture questions, and an offer evaluation framework. The candidate should understand which remote opportunities are genuinely compatible with their location, schedule, and career goals.

The workflow should reduce wasted applications to vague remote listings and increase conversations with teams that actually operate well in distributed environments.
