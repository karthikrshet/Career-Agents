# Internship Hunt

# Overview

Internship Hunt is a complete operating system for students pursuing competitive internships. It combines profile building, target selection, application packaging, deadline management, outreach, assessment preparation, and weekly review. The workflow assumes the student may have limited formal experience, but it does not accept "I have nothing" as the final answer. Coursework, projects, hackathons, clubs, volunteering, open-source work, freelance tasks, and self-directed learning can all become evidence if framed honestly.

The objective is to create a repeatable internship pipeline rather than a last-minute application scramble. Students should know which programs matter, when they close, what materials are required, who can help, what assessments are likely, and what weekly work keeps the search alive.

# Target Audience

- Undergraduate students seeking summer, winter, or semester internships.
- Final-year students who need internship-to-placement leverage.
- Freshers trying to build experience before full-time applications.
- Students applying through campus portals, company programs, referrals, or direct applications.
- Contributors designing early-career workflows for Career-Agents.

# Prerequisites

- Current resume or rough resume draft.
- Academic year, graduation date, degree, and eligibility constraints.
- List of projects, coursework, competitions, leadership roles, and skills.
- Target industries or role families.
- Internship season and deadline calendar.
- LinkedIn profile.
- Portfolio, GitHub, design portfolio, writing samples, or research artifacts if relevant.

# Step-by-Step Execution Plan

1. **Define the internship target.** Choose 1-3 role families, such as software engineering intern, data analyst intern, product intern, research intern, design intern, or marketing intern. Avoid applying to unrelated roles with the same resume.

2. **Check eligibility.** Record graduation year, academic year, location, work authorization, GPA requirements, program restrictions, and required documents. Eliminate programs the student cannot legally or academically enter.

3. **Mine student evidence.** Build an evidence inventory from projects, coursework, labs, hackathons, clubs, volunteering, part-time work, teaching assistance, writing, and personal builds. Each item should show skill, action, and result.

4. **Build the internship resume.** Use `resume-strategist` and `ats-resume-reviewer` to create a one-page resume with education, skills, projects, experience, and achievements ordered for the target role.

5. **Create project explanation scripts.** For every resume project, prepare a 60-second explanation: problem, contribution, tools, challenge, result, and learning.

6. **Build the target matrix.** List companies, programs, deadlines, eligibility, role fit, application channel, required materials, and assessment format. Use reach, match, and safety tiers.

7. **Prepare application packages.** For high-priority internships, tailor resume summary, project order, cover letter or written responses, and portfolio links.

8. **Launch outreach.** Contact alumni, campus seniors, recruiters, professors, community leads, or employees with concise messages. The ask should be specific: referral guidance, application advice, or a short conversation.

9. **Submit in batches.** Apply weekly, not randomly. Each batch should include a mix of reach, match, and safety opportunities.

10. **Prepare for assessments.** Identify whether the internship uses coding tests, case interviews, portfolio reviews, video responses, HR screens, or take-home tasks. Build practice blocks before invitations arrive.

11. **Review and iterate.** Every week, check applications submitted, responses, assessments, interviews, rejections, and deadlines missed or approaching.

# Weekly Action Plan

**Week 1: Foundation**

- Build evidence inventory.
- Finalize role families.
- Create resume v1.
- Audit LinkedIn profile.
- Build first 30-target internship matrix.

**Week 2: Application Package**

- Tailor resume for top two role families.
- Write reusable cover-letter blocks.
- Prepare project scripts.
- Submit 8-12 priority applications.
- Send 10 targeted outreach messages.

**Week 3: Outreach and Assessments**

- Expand target matrix to 50 opportunities.
- Continue applications.
- Practice likely assessment format.
- Follow up on previous outreach.
- Improve weak project explanations.

**Week 4: Response Review**

- Calculate response rate by role family.
- Adjust resume if no responses.
- Prioritize interviews and assessments.
- Add backup opportunities.
- Prepare next-month sprint.

# Daily Checklist

- Check internship deadlines.
- Update application tracker.
- Tailor one application before submitting.
- Send or follow up on two outreach messages.
- Practice one assessment task or project explanation.
- Improve one resume bullet or portfolio note.
- Save recruiter feedback or rejection data.
- Avoid applying to roles where eligibility is not met.

# KPIs / Success Metrics

- Priority applications submitted before deadline.
- Application-to-response rate.
- Outreach response rate.
- Referrals or warm introductions generated.
- Assessment invitations received.
- Interviews booked.
- Number of projects with strong explanation scripts.
- Percentage of applications tailored by role family.

# Common Mistakes

- Waiting until deadlines are close.
- Saying "no experience" and ignoring project evidence.
- Applying to every internship with the same generic resume.
- Missing eligibility requirements.
- Failing to prepare for assessments before invitations.
- Asking alumni for referrals without showing fit.
- Listing projects without being able to explain them.
- Applying only to famous companies and ignoring strong learning opportunities.

# Expected Outcomes

By the end of this workflow, the student should have an internship-ready resume, a role-specific evidence inventory, a target matrix, application tracker, outreach sequence, project explanation scripts, and assessment plan. The student should know which opportunities are high priority, which are backups, and which are not worth time.

The workflow should leave the student with a system they can run weekly until the internship is secured: update targets, submit batches, follow up, prepare assessments, review outcomes, and improve materials.
